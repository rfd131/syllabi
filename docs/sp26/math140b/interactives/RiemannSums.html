Notes
Why this works and what the picture is showing.

Riemann sum, formally

Partition [a, b] into n equal pieces of width Δx = (b − a)/n. Pick a sample point xi* in each subinterval [xi, xi+1]. The Riemann sum is

Sn = Σ f(xi*) · Δx.

When f is continuous, every choice of sample points gives the same limit as n → ∞. That common limit is defined to be ∫ab f(x) dx.

Signed area

If f(xi*) is negative, the rectangle contributes a negative number to the sum — that is signed area. Try sin(x) on [0, 2π]: the positive bump on [0, π] and the negative bump on [π, 2π] cancel, so the integral is 0.

Left & right bracket the integral (monotonic case)

If f is increasing on [a, b], the left rule underestimates the integral and the right rule overestimates it — the exact value is trapped between them. If f is decreasing, the roles swap. Either way, |Ln − Rn| gives a crude error bound.

Why midpoint & trapezoidal converge faster

For smooth f, left/right errors are O(1/n) because each rectangle misses a triangular sliver of size ≈ f′·Δx². Midpoint and trapezoidal cancel the linear piece of the error, leaving O(1/n²). Simpson's rule matches quadratics exactly and achieves O(1/n⁴) for smooth integrands.

On the log–log plot on the previous card, convergence order shows up as the slope: rules with steeper slopes converge faster.
