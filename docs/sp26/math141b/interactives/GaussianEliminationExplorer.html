Gaussian elimination explorer

Pick a linear system, then reduce its augmented matrix by typing elementary row operations. Each row operation corresponds to a legal manipulation of the underlying equations — the solution set is always preserved. Watch the equations, the matrix, and the operation history stay in sync.

Example systems

Syntax. Rows are R1, R2, …. Numbers can be integers or fractions like 3/4 or -1/2. Arrow -> assigns; <-> swaps.

Row operation ↔ equation correspondence
The three elementary row operations each map to a manipulation of the underlying equations that preserves the solution set.

Row swap

Ri <-> Rj

Reordering two equations. Equation i and equation j trade places.

The set of equations is unchanged — only their order changes — so the solution set is identical.

before: x + 2y = 5 3x + y = 5 after: 3x + y = 5 x + 2y = 5

Row scaling

Ri -> c · Ri (c ≠ 0)

Multiplying an equation by a nonzero constant. Both sides of equation i are scaled.

An equation and any nonzero multiple of it have the exact same solutions, so the solution set is preserved. c = 0 is not allowed — it would erase the equation.

before: 2x + 4y = 6 after (× ½): x + 2y = 3

Row replacement

Ri -> Ri + c · Rj

Adding a multiple of one equation to another. Replace equation i with equation i plus c times equation j.

If both original equations hold, any linear combination of them also holds — so the new equation is implied by the originals, and the solution set does not change. This is the workhorse of Gaussian elimination: it introduces zeros below pivots.

R1: x + 2y = 5 R2: 3x + y = 5 R2 -> R2 - 3·R1: 0x - 5y = -10

Accepted input forms

  • R1 <-> R2 — swap two rows
  • R1 -> 3*R1, R2 -> (1/2)R2, R3 -> -R3 — scale
  • R2 -> R2 - 3*R1, R3 -> R3 + (2/5)*R1 — replacement
  • R2 - 3*R1 -> R2 — textbook-style form (expression on the left) is also accepted

The * between a number and a row is optional. Rows are case-insensitive: r1, R1, Row 1 all work.
